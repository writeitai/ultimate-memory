1"""The relation supersession cascade (D3/D4): blocking → novelty → ladder. 

2 

3Adjudication operates on RELATIONS, never claims (D3): "Alice left Acme" 

4closes the validity window of `(alice, works_for, acme)` — one row update — 

5while every claim stays an immutable record of what sources asserted. 

6Candidates are found by `(subject, predicate)` blocking over the small 

7distinct-fact table, only for change-prone predicates; a novelty gate routes 

8clear ADDs past the LLM entirely; the ambiguous residue climbs the 

9small→frontier ladder. Every decision lands append-only in 

10`relation_adjudications` — the S8 "why do we believe…" audit surface — and 

11the adjudicator fails safe to coexist when unsure. 

12"""

35_ADJUDICATION_PROMPT: Final = """You adjudicate fact supersession for a memory 

36system. Two believed facts share a subject and a change-prone predicate: 

37 

38EXISTING: {existing_label} 

39 evidence: {existing_evidence!r} (asserted {existing_asserted}) 

40NEW: {new_label} 

41 evidence: {new_evidence!r} (asserted {new_asserted}) 

42 

43Decide: 

44- supersede: the world changed — the NEW fact replaces the EXISTING one 

45 (e.g. a job change); the existing fact's validity window should close. 

46- coexist: both hold simultaneously (e.g. dual employment). When unsure, 

47 prefer coexist — a wrong supersession silently hides a true fact. 

48- contradict: the sources describe the SAME period incompatibly; both must 

49 stand, surfaced together."""

185 new_relation_id: UUID,

339_LOAD_RELATION = text( 

340 """ 

341 SELECT r.relation_id, r.subject_entity_id, r.predicate, r.object_entity_id, 

342 r.invalidated_at, r.contradiction_group, 

343 coalesce(r.fact_label, 

344 subject.canonical_name || ' ' || r.predicate || ' ' 

345 || object.canonical_name) AS label, 

346 p.is_change_prone, 

347 evidence.claim_text AS evidence_text, 

348 evidence.asserted_at 

349 FROM relations r 

350 JOIN predicates p ON p.deployment_id = r.deployment_id 

351 AND p.predicate = r.predicate 

352 JOIN entities subject ON subject.entity_id = r.subject_entity_id 

353 JOIN entities object ON object.entity_id = r.object_entity_id 

354 LEFT JOIN LATERAL ( 

355 SELECT c.claim_text, c.asserted_at 

356 FROM relation_evidence e 

357 JOIN claims c ON c.claim_id = e.claim_id 

358 WHERE e.relation_id = r.relation_id AND e.stance = 'supports' 

359 ORDER BY c.ingested_at DESC 

360 LIMIT 1 

361 ) evidence ON true 

362 WHERE r.deployment_id = :deployment_id AND r.relation_id = :relation_id 

363 """ 

364) 

365 

366_BLOCK_CANDIDATES = text( 

367 """ 

368 SELECT r.relation_id, r.object_entity_id, r.contradiction_group, 

369 coalesce(r.fact_label, 

370 subject.canonical_name || ' ' || r.predicate || ' ' 

371 || object.canonical_name) AS label, 

372 evidence.claim_text AS evidence_text, 

373 evidence.asserted_at 

374 FROM relations r 

375 JOIN entities subject ON subject.entity_id = r.subject_entity_id 

376 JOIN entities object ON object.entity_id = r.object_entity_id 

377 LEFT JOIN LATERAL ( 

378 SELECT c.claim_text, c.asserted_at 

379 FROM relation_evidence e 

380 JOIN claims c ON c.claim_id = e.claim_id 

381 WHERE e.relation_id = r.relation_id AND e.stance = 'supports' 

382 ORDER BY c.ingested_at DESC 

383 LIMIT 1 

384 ) evidence ON true 

385 WHERE r.deployment_id = :deployment_id 

386 AND r.subject_entity_id = :subject_entity_id 

387 AND r.predicate = :predicate 

388 AND r.relation_id <> :relation_id 

389 AND r.invalidated_at IS NULL 

390 AND (r.valid_until IS NULL OR r.valid_until > now()) 

391 ORDER BY r.ingested_at 

392 """ 

393)
